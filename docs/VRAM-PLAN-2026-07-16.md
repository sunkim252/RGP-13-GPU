## 항목
- **P1 dvec/d3 [3nf ×2 = 468 MB] — 완료 확인**: NO-src 디바이스 아밍이 rgpSTDvecRelease 호출(프로덕션 로그 확인). 리스크 0.
- **P2 grad9 [9nc = 190 MB] — 실행**: UEqn grad9는 momentumPredictor 국면에서만 살아 있고, 매니폴드 SoA(gB.out ≥ 17nc)는 그 국면에 죽어 있음(모든 소비자 = updateManifold/thermo-chain 내부; 다음 evaluate가 전체 재기록). → grad9를 SoA 버퍼에서 차용. gmc 모듈/미아밍 폴백 = 자체 할당 유지.
  - 스케줄 불변식(적대 리뷰로 교정): grad9 클로버는 momentum뿐 아니라 **pCorr의 NOCorrPrep kf/Cf 스테이징에서도** 발생. 실제 안전 근거 = "모든 SoA 읽기 직전에 같은 corrector 시퀀스에서 evaluate 전체 재기록이 선행"(per-corrector updateManifold). 이 순서 계약은 rgpFgmScratchDirty()(차용 기록 시 lastN=0 fail-closed)로 강제 — 위반 시 무음 오염 대신 명시 에러.
- **P4 gRW.out [106×nSub = 103 MB] — 실행**: SoA-생략 모드의 Y-보간 D2H를 ≤16필드 청크로 (버퍼 16×nSub=16 MB).
- **P3 (보류, 문서화) 위상-분리 셀 스크래치 풀 [~210 MB]**: gST(tp 국면) ↔ gB pEqn(pCorr 국면) 별칭. M4 pOld/rdt 스탬프 무효화 결합 필요 — 클로버-무음-오답 클래스 리스크 대비 이득 0.08 GB/M로 보류.
- **불가 판정**: 정적 지오메트리 별칭(재업로드가 상주보다 비쌈), FP32(비트-계약), 테이블 랭크 공유(WSL IPC 거부 — 네이티브는 이미 IPC로 해결).
