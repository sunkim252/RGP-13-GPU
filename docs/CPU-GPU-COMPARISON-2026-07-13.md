**케이스**: rd0110 2Mv2 — 2,604,771셀, SRK 106종 리얼가스 fgm 풀스택
(LOX/케로신 동축 인젝터, 비직교 스냅 메시, LTS). 동일 코드베이스
(W-시리즈 최적화 반영 HEAD — Deff 캐시 등 CPU 경로 개선도 포함된

## 해석

1. **핵심 답: CPU 8코어 대비 GPU 8랭크(PRO 6000)가 8.3× 빠름.**
   같은 8-way 병렬 자원끼리의 비교로, 노드 관점에서 GPU 1장 추가가
   CPU-only 대비 스텝 시간을 72.4→8.7s로 줄인다.
2. **같은 머신(parcom5) 내 비교**: 구형 6GB 1660S **1장(직렬)**이
   Xeon 8코어보다 1.2× 빠름 — GPU 오프로드된 부분(thermo/manifold/
   U/P/ZC 솔브)이 CPU에서 얼마나 비쌌는지의 직접 증거.
3. **비대칭의 구조**: GPU 빌드의 병목은 이제 호스트 잔여(nsys 실측:
   PRO 6000에서 GPU 커널 0.94s + 전송 1.9s, 나머지 호스트)라
   GPU 랭크 확장은 8에서 정체. CPU-only는 코어 수에 계속 스케일
   가능하나, 8.3× 격차를 코어로 메우려면 ~64+ Xeon 코어가 필요한
   수준(통신 효율 무시 시)이다.
4. **주의**: 머신이 다른 행(4060/PRO6000)은 CPU 세대 차이가 섞임.
   같은-머신 순수 비교는 parcom5 행(72.4 vs 59.0)이 유일하며,
   CPU 행 자체는 Xeon 6248R 기준이라 TRX50에서 CPU-only를 돌리면
   더 빠를 수 있음(사용자 지시로 GPU 머신에서는 CPU 실측 미수행).
   로컬(WSL) CPU 8랭크는 초기화+스텝1 진입까지 확인 후 지시로 중단.
